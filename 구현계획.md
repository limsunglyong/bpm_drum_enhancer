### Phase 1 — 오디오 기능 (2026-05-25 완료)
| 항목 | 구현 내용 |
|---|---|
| **Pan (스테레오 패닝)** | `drum-engine.js` — trigger()에 pan 파라미터 추가, StereoPannerNode 생성. `app.jsx` — trigger 호출부에 `t.pan` 전달 |
| **Loop 제어 (1회 재생)** | `app.jsx` — useScheduler에 `loop` ref 추가. `loop=false`시 마지막 스텝 이후 자동 stop |
| **Metronome (클릭 사운드)** | `drum-engine.js` — `metroClick(t, isDownbeat)` 추가 (1500Hz/1000Hz sine, 30ms decay). `app.jsx` — beat start마다 호출 |
| **Master Buss 스펙트럼 (실제 FFT)** | `drum-engine.js` — `getSpectrum()` 추가 (AnalyserNode frequencyBinCount Uint8Array). `components.jsx` — MasterCard에 로그 스케일 32-bar FFT 시각화 |

### Phase 2 — 데이터 저장 / 로드 (2026-05-25 완료)
| 항목 | 구현 내용 |
|---|---|
| **Pattern Bank (A/B/C/D)** | `app.jsx` — localStorage `fdaw_slot_X` 키에 직렬화 저장/로드. 빈 슬롯 클릭 → 현재 패턴 저장, 채워진 슬롯 클릭 → 패턴 로드. "Update Slot X" 버튼으로 현재 슬롯 갱신 |
| **Save to Library** | `app.jsx` — localStorage `fdaw_library` 배열에 저장 (최대 50개). 이름 입력 모달 연동 |

### Phase 3 — Export (2026-05-25 완료)
| 항목 | 구현 내용 |
|---|---|
| **Export WAV** | `app.jsx` — `DrumEngine.renderToBuffer()` (OfflineAudioContext 48kHz). `audioBufferToWav()` 16-bit PCM 인코더. 파일 다운로드 |
| **Export MIDI** | `app.jsx` — `patternToMidi()` SMF Type-0, TPQN=480, GM 드럼 채널(ch 10), VLQ 인코딩. GM 노트맵: kick=36, snare=38, hihat=42, openhat=46, clap=39, tom=41, perc=75, rim=37 |
| **Export Stems** | `app.jsx` — 트랙별 `DrumEngine.renderTrackToBuffer()`. JSZip(CDN)으로 ZIP 생성. `drum-beat-maker.html`에 JSZip CDN 추가 |
| **Export Focustone Pattern (.fpatt)** | `app.jsx` — 전체 state JSON 직렬화 (version/name/tracks/pattern/bpm/settings). `.fpatt` 확장자 다운로드 |
| **drum-engine.js render API** | `renderToBuffer()`, `renderTrackToBuffer()` 추가. 모든 voice 함수가 `(t, vel, dest, ctx)` 파라미터 수용 (OfflineAudioContext 주입 지원) |

### Phase 4 — UX / 입력 확장 (2026-05-25 완료)
| 항목 | 구현 내용 |
|---|---|
| **Keyboard Shortcuts** | `app.jsx` — `useEffect` keydown 핸들러. Space(재생/정지), Esc(정지), R(녹음), L(루프), M(뮤트), S(솔로), ↑↓(트랙 이동) |

## 🔧 Phase 6 — 레이아웃 정렬 버그 수정

> 원인 1: BAR ruler spacer(`w-[252px]`)와 TrackRow 패드 시작점(`px-2 8px + label 252px + gap-3 12px = 272px`) 사이에 **20px 불일치**. spacer를 `w-[272px]`로 수정.
> 원인 2: BAR ruler 컬럼 폭 공식이 스텝 간 gap 누락 — `stepsPerBar×minCellPx`만 사용해 PAD 실제 폭(`stepsPerBar×minCellPx + (stepsPerBar-1)×cellGap`)과 불일치.

| 항목 | 수정 내용 | 수정 파일 | 상태 |
|---|---|---|---|
| **BAR ruler ↔ PAD 정렬 오류** | spacer `w-[252px]`→`w-[272px]`, pr-2 추가, minWidth `252+N`→`280+N`. 추가로 BAR 컬럼 폭 공식에 내부 gap 포함: `minCellPx×stepsPerBar + (stepsPerBar-1)×cellGap` | `app.jsx` | ✅ 완료 |
| **선택 트랙 배경 좌우 정렬 오류** | 위 BAR ruler spacer 수정으로 동시 해결 | `app.jsx` | ✅ 완료 |
| **수평 스크롤바 위치·크기** | bar-scroll/pad-scroll 2분할 시도했으나 JS 동기화 시 sticky 재계산 불안정으로 레이블 이동 버그 발생. 단일 `overflow-x-auto`로 복원. 스크롤바는 트랙 목록 하단에 위치 (별도 개선 필요) | `app.jsx`, `drum-beat-maker.html` | ⬜ 보류 |
